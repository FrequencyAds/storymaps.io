For Developers

A CLI that fits your workflow

Install via NPM and manage your story maps from the terminal. Push, pull, diff - just like git. Works with any self-hosted instance.

npm install -g storymaps View on npm
$ storymaps init# Initialise a local map
$ storymaps push# Push with conflict detection
$ storymaps pull# Pull latest from remote
$ storymaps diff# Summary or line-by-line diff
$ storymaps status# Progress bars per slice
$ storymaps log# Edit history

FAQ

Frequently asked questions

Is Storymaps.io really free?
Yes, completely. Storymaps.io is licensed under AGPL-3.0 and is free to use with no limits on maps, collaborators, or features. There are no paid plans and no feature gates.
Do I need to create an account?
No. Just open storymaps.io and start mapping. There is no signup, no login, and no account required. Your maps are saved automatically and accessible via their unique URL.
What is user story mapping?
User story mapping is a technique popularised by Jeff Patton for organising user stories into a visual map. Activities run across the top, and stories are arranged vertically by priority, making it easy to see the big picture, plan releases, and identify gaps in the user experience.
Can I export to Jira?
Yes. You can export your stories directly to Jira, Asana, Linear, and Phabricator. You can also export and import your maps as JSON, YAML, or CSV for maximum flexibility.
Is my data safe?
Your maps are stored on our server and can be exported at any time as JSON or YAML. The project is open source, so you can also self-host it on your own infrastructure for full control over your data.
Can I self-host it?
Yes. Storymaps.io ships with a Docker setup and is licensed under AGPL-3.0. Clone the repository, run Docker Compose, and you have your own instance. See the GitHub repository for instructions.
